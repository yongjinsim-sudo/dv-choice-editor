# DV Choice Editor

Dataverse choice management inside VS Code.

**DV Choice Editor** is a focused utility from **DV ForgeLab** for managing Dataverse local choice columns and global choices directly from VS Code.

The extension provides a lightweight, preview-first workflow for creating, updating, deleting, reviewing, and publishing Dataverse choice values without leaving the development workspace.

---

## v1.4.0 — Native Reconstruction Artifacts

DV Choice Editor now supports both local entity choice columns and global choices through the same preview-first workflow.

Supported local/global operations:

* Load choice values
* Add values
* Update labels
* Delete values
* Preview staged metadata changes
* Apply and publish explicitly
* Export native DVCE `.json` choice definition artifacts
* Import native DVCE `.json` artifacts
* Import DV Quick Run `.dvce.json` reconstruction artifacts using the same DVCE-owned schema

DV Choice Editor remains the owner of the `dvce.choiceDefinition` artifact schema.

Manual exports generated by DV Choice Editor use standard `.json` files:

* `dvce-{entity}-{attribute}-{yyyyMMdd-HHmmss}.json`
* `dvce-{globalChoiceName}-{yyyyMMdd-HHmmss}.json`

DV Quick Run reconstruction handoff artifacts use the `.dvce.json` extension while preserving the same native DVCE schema, allowing DVCE to distinguish reconstruction artifacts from standard exports without introducing a separate artifact format.

Legacy v1.x/v2.x value-array artifacts remain fully supported.

### JSON Definition Artifacts

DV Choice Editor exports native DVCE choice definition artifacts as standard `.json` files and can import both standard `.json` exports and DV Quick Run `.dvce.json` reconstruction artifacts using the same `dvce.choiceDefinition` schema.

#### Local Choice Artifact

```json
{
  "artifactType": "dvce.choiceDefinition",
  "version": "3.0",
  "scope": "local",
  "entityLogicalName": "account",
  "attributeLogicalName": "new_membershiptype",
  "values": [
    {
      "label": "Gold",
      "value": 100000000
    }
  ]
}
```

#### Global Choice Artifact

```json
{
  "artifactType": "dvce.choiceDefinition",
  "version": "3.0",
  "scope": "global",
  "optionSetName": "new_membershiptype",
  "values": [
    {
      "label": "Gold",
      "value": 100000000
    }
  ]
}
```

Imported definitions are compared against the currently loaded target. Missing values and label changes are staged locally, then reviewed through the normal preview-first workflow before publishing.

### Preview-First Workflow

All metadata changes are staged locally before being applied.

```text
Select local choice column
or
Select global choice
↓
Stage changes
↓
Review preview
↓
Apply and publish
```

## DVCE v3.0 Operation Artifacts

DVCE v1.4.0 accepts native operation-based `.json` / `.dvce.json` artifacts. This is the preferred reconstruction handoff format because DVCE remains the artifact owner and the workflow still follows:

```text
Import
↓
Stage
↓
Validate
↓
Preview
↓
Apply
```

```json
{
  "artifactType": "dvce.choiceDefinition",
  "version": "3.0",
  "generatedBy": "DV Quick Run",
  "generatedUtc": "2026-06-26T00:00:00.000Z",
  "scope": "local",
  "entityLogicalName": "account",
  "attributeLogicalName": "new_status",
  "displayName": "Status",
  "operations": [
    { "operation": "AddOption", "label": "Pending Review", "value": 100000001 },
    { "operation": "UpdateLabel", "value": 100000002, "nextLabel": "Approved" },
    { "operation": "DeleteOption", "value": 100000003, "label": "Deprecated" }
  ]
}
```

Supported operation names:

* `AddOption`
* `UpdateLabel`
* `DeleteOption`

When a VS Code workspace is available, DVCE imports from and exports to `.dvforgelab/dvce/exports`. Native DVCE exports use the .json extension, while DV Quick Run reconstruction artifacts use .dvce.json.
